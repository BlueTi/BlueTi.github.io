소개

생각을 표현하는 도우미

  • 글쓰기에 편리한 도구제공
    • 평소 글을 쓰다 표현하기 어렵거나 뒷 내용의 매끄러운 전개에 어려움을 겪을때 AI가 학습한 내용을 기반을 뒤의 내용을 연결하는 문장들을 추천하여 도움
    • 장시간 글쓰기로 피곤해 질 수 있는 집필 환경에 대한 환경 도움
      • 다크모드를 이용한 눈의 피로 저하
      • 챕터 분리 표기 및 저장으로 소주제에 대한 집중
      • 작가의 취향에 맞는 폰트 스타일 제공
      • 내용에 도움이 될 수 있는 자료들을 기입,표시하는 메모 기능

다양한 생각을 접하는 만남의 장

  • 다양한 작가들의 글을 검색 또는 추천으로 만남
    • 메인 페이지에서 많은 사람들이 보는 게시글 추천
    • 로그인 하였을 시 사용자의 선택 경향에 따른 글 추천
    • 태그를 이용한 글 검색

아키텍처 구조

담당업무

  • Spring Boot REST API 구현
  • 보안처리를 위해 Vue Client에서 AWS Cognito를 통해 인증받은 jwt Token을 확인하고 API에 접근하도록 구현
  • 가상서버 확장에 용이하도록 AWS S3에 필요한 이미지 데이터들을 전송,참고하도록 구현
  • 개발 시 Debian OS + Docker + MongoDB를 이용하여 개발 중인 팀원들이 구현한 서비스를 확인하도록 구현
  • AI 학습 및 모델 선정을 위한 검토
  • Docker + Travis CI를 이용한 AWS ElasticBeanStalk 배포 이용

사용 기술

  • OS : Ubuntu
  • Language : Python, Java
  • Framework: Vue, Flask ,TensorFlow ,Spring
  • DB : MongoDB
  • Cloud : AWS(EC2, S3, Cognito,ElasticBeanStalk)

프로젝트 총평

배운 점

  • Tensorflow + Cuda를 이용한 AI 모델 학습 및 이용
  • Spring, AWS, Docker가 제공하는 강력한 서비스 생산 및 배포 기능
    • Backend 개발에서 Spring Restful API+ Vue의 이용으로 상당히 강력하고 빠르게 실사용 가능한 서비스를 빠르게 제작할 수 있음에 놀라웠다.
    • Spring에서 AWS는 물론 다양한 서비스들에 연결을 이미 제공, 1번은 가져다 쓰는 법을 아는 것이지만 더 큰 제어를 위해서 기술 기반의 작동 원리도 알아야 했다.
    • 개발 중 Travis CI가 깃허브를 관찰하여 내가 푸쉬한 내용들을 자동으로 빌드하여 배포하는 과정에서 많은 오류가 있었지만 도커 이미지 생성을 위한 명령어 체계
  • 수업도 좋지만 직접 개발의 성장 속도 차이
  • 협업
    • 좋은 협업을 위해선 뚜렷한 역할 선정이 중요
    • 분리된 역할에 대한 최선
    • 다른 팀원의 역할 이해가 더 해지면, 더 좋은 커뮤니케이션과 작업이 가능
    • 토론은 늘 자료와 데이터 기반
    • 상대에 대한 배려는 장기간 큰 효과
